# Типовое проектирование информационных систем

## 1. Основные понятия типового проектирования

Методы типового проектирования информационных систем предполагают создание ИС из **готовых типовых элементов** — типовых проектных решений (ТПР).

Для применения типового проектирования проектируемая ИС должна быть **декомпозируема** на составляющие компоненты:
- подсистемы;
- программные модули;
- комплексы задач и др.

**Типовое проектное решение (ТПР)** — это представленное в виде проектной документации (включая программные модули) проектное решение, пригодное к **многократному использованию**.

---

## 2. Методы типового проектирования

Выделяют три основных метода типового проектирования ИС:
1. элементный метод;
2. подсистемный метод;
3. объектный метод.

---

## 3. Элементный метод типового проектирования

При элементном методе в качестве типового элемента системы используется:
- типовое решение по задаче;
- либо по отдельному виду обеспечения задачи:
  - информационному;
  - программному;
  - техническому;
  - математическому;
  - организационному.

### Достоинства элементного метода
- модульный подход к проектированию;
- модульный подход к документированию ИС.

### Недостатки элементного метода
- большие затраты времени на сопряжение разнородных элементов из-за:
  - информационной несовместимости;
  - программной несовместимости;
  - технической несовместимости ТПР;
- плохая адаптивность элементов к особенностям предприятия.

Из-за больших затрат времени на доработку отдельных элементов ТПР имеют **ограниченный характер применения**.

Элементные ТПР применяются в основном:
- в виде библиотек методо-ориентированных программ;
- библиотек классов объектов;
- при разработке графических интерфейсов;
- при использовании вычислительных и служебных функций.

---

## 4.Подсистемный метод типового проектирования

При подсистемном методе типовым элементом системы являются **отдельные подсистемы**.

ТПР для функциональных подсистем реализуются в виде **пакетов прикладных программ (ППП)**.

Подсистемный метод позволяет:
- осуществлять модульное проектирование;
- выполнять параметрическую настройку программных компонентов под различные объекты управления;
- сокращать затраты на проектирование и программирование взаимосвязанных компонентов;
- обеспечивать хорошее документирование процессов обработки информации.
- 
### Достоинства подсистемного метода
- параметрическая настраиваемость подсистем;
- минимизация внешних информационных связей;
- более высокая степень интеграции типовых элементов ИС.

### Недостатки подсистемного метода
- проблемы при объединении ППП разных функциональных подсистем;
- информационная, программная и техническая несовместимость ППП;
- при построении единой ИС, особенно в случае использования ППП нескольких производителей программного обеспечения.
---

## 5.Объектный метод типового проектирования

При объектном методе типовым элементом является **типовой проект для объектов управления определённой отрасли**, включающий:
- полный набор функциональных подсистем;
- полный набор обеспечивающих подсистем ИС.

### Характерные черты типовых проектов
- **Открытость архитектуры** — возможность установки на различных программно-технических платформах;
- **Масштабируемость** — конфигурация ИС для различного числа рабочих мест;
- **Конфигурируемость** — выбор подмножества компонентов и их параметрическая настройка под особенности объекта управления.

### Достоинства объектного метода
- информационная совместимость компонентов;
- программная совместимость компонентов;
- техническая совместимость компонентов.

### Недостатки объектного метода
- необходимость адаптации типового проекта к конкретному объекту управления

---

## 6.Параметрически-ориентированное проектирование

При параметрически-ориентированном проектировании пакет прикладных программ (ППП) рассматривается как **«чёрный ящик»**.

### Общая схема параметрически-ориентированного проектирования

На вход ППП поступают два вида потоков:
- **информационный поток**;
- **параметрический поток**.

Внутри ППП выделяются следующие блоки:
- **блок функционирования** — реализует основные функции системы;
- **блок обработки параметров** — интерпретирует значения параметров;
- **блок адаптации** — обеспечивает настройку и модификацию работы системы.

На выходе формируются **результаты работы** системы.

```
     ПАРАМЕТРИЧЕСКИЙ ПОТОК       ИНФОРМАЦИОННЫЙ ПОТОК
               |                          |
               v                          v
        -------------------------------------------------
        |                                               |
        |        ПАКЕТ ПРИКЛАДНЫХ ПРОГРАММ (ППП)         |
        |                                               |
        |                +-----------------+            |
        |                |     Блок        |            |
        |                | функционирования|            |
        |                +--------+--------+            |
        |                         ^                     |
        |                         |                     |
        |        +----------------+----------------+    |
        |        |                                 |    |
        | +------+-------------+       +-----------+--+ |
        | |  Блок обработки    |       |   Блок       | |
        | |    параметров      |       |  адаптации   | |
        | +--------------------+       +--------------+ |
        |                                               |
        -------------------------------------------------
                          |
                          v
                  РЕЗУЛЬТАТЫ РАБОТЫ


```
---

### Параметрический поток

**Параметрический поток** — это информация, необходимая для настройки пакета прикладных программ на конкретные условия функционирования.

Параметрический поток включает информацию:
- которая задаётся **один раз при установке (инсталляции)** ППП;
- с помощью изменения параметров можно:
  - включать и выключать отдельные модули;
  - изменять режимы работы модулей;
  - адаптировать систему под особенности объекта управления.

---

### Критерии выбора параметрически-ориентированного проектирования

При выборе параметрически-ориентированного проектирования учитываются следующие факторы:
- срок разработки информационной системы;
- наличие финансовых ресурсов;
- техническая оснащённость объекта управления;
- существующие функционирующие ППП;
- программное и сетевое оснащение;
- другие организационные и технические факторы.

---

### Проблемы, сдерживающие распространение параметрически-ориентированного проектирования

Основными проблемами являются:
- психологические и организационные трудности внедрения ППП;
- высокая стоимость приобретения ППП и обучения персонала;
- отсутствие глобальной модели объекта управления, что приводит к:
  - дополнительным затратам на согласование;
  - необходимости увязки различных ППП в рамках одной ИС.
