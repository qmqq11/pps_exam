# 1 ОТС + ИС

## Общая теория систем (ОТС)

**Система** — целостность взаимозависимых элементов;  
при разбиении теряются свойства системы и элементов.

### Характеристики системы
- **эмерджентность** — свойства системы ≠ сумма свойств элементов;
- собственные закономерности функционирования;
- развитие во времени.

### Элементы системы
- **элемент** — неделимая часть системы;
- **связь** — отношения между элементами;
- **структура** — необходимые и достаточные связи, определяющие свойства системы.

### Состав системы
**Вход → Процесс → Выход**

**Вход включает:**
- объект воздействия (сырьё, энергия, информация);
- внешнюю среду (случайные факторы, внешние целенаправленные системы);
- управляющие воздействия (цели, приказы, ограничения).

**Процесс** — преобразование входа.  
**Выход** — результат деятельности (цели, стабильность, надёжность).

### Обратная связь
Передача сигнала с выхода на вход для управления.

**Функции:**
- компенсация возмущений;
- поддержание устойчивости;
- корректировка отклонений;
- формирование управляющих воздействий.

Нарушение обратной связи → потеря адаптации и развития.

### Классификация систем
- по природе: живые / неживые;
- по элементам: физические / абстрактные;
- по происхождению: естественные / искусственные;
- по составу: машинные / человеко-машинные / социальные;
- по целям: одноцелевые / многоцелевые / функциональные;
- по среде: открытые / закрытые  
(все живые — открытые).

### Устойчивость и адаптация
- **устойчивость** — сохранение параметров;
- **неустойчивость** — потеря функций;
- **ультрастабильность** — устойчивость через изменения.

**Адаптация:**
- пассивная — реакция;
- активная — воздействие на среду.

**Самонастройка** — изменение функционирования.  
**Самоорганизация** — изменение структуры.

### Большие и сложные системы
- **большая** — много однотипных элементов, иерархия;
- **сложная** — разнородные элементы и связи.

### Система управления
Воздействие на объект для изменения его состояния и процессов.

---

## Информационные технологии и системы (дополнительно)

**ИС** — обработка информации + ресурсы  
(человеческие, технические, финансовые).

**Программная система** — ПО + данные + аппаратура  
(ценность создаёт ПО).

**АС** — автоматизация информационных процессов
при участии человека и техники.

### Информационное обеспечение (ИО)
Данные, классификаторы, документы, формы, интерфейсы.

**Назначение ИО:**
- кодирование информации;
- анализ и обработка;
- взаимодействие пользователей;
- управление деятельностью.

### Информационные базы
- **внемашинная ИБ** — документы, классификаторы;
- **внутримашинная ИБ** — данные на носителях.

**БД** — структурированный набор данных для функций ИС.

### Состав ИС
- **функциональные подсистемы** — решают задачи;
- **обеспечивающие подсистемы** — поддерживают работу ИС
(ИО, ПО, ТС, орг., прав., матем., лингв.).

### ГОСТ (кратко)
ИО должно быть:
- достаточным;
- совместимым;
- стандартизированным;
- защищённым;
- контролируемым.






# Предпроектное обследование ИС

## Предпроектное обследование
Начальный этап создания ИС.  
Цель — понять объект автоматизации, проблемы, цели и целесообразность ИС,
согласовать ожидания заказчика и исполнителя.

---

## Общие сведения о предприятии
Название, вид деятельности, сфера, положение на рынке.  
Нужны для понимания контекста проекта.

---

## Цели и задачи предприятия
Цели — желаемые результаты.  
Задачи — действия для их достижения.  
Фиксируются KPI для оценки эффективности.

---

## Организационная структура
Состав подразделений, уровни управления, подчинённость.  
Используется для определения пользователей ИС и границ автоматизации.

---

## Группировка в оргструктуре
- **Функциональная** — по функциям (бухгалтерия, маркетинг).
- **Рыночная** — по продуктам, клиентам или регионам.

---

## Типы организационных структур
- **Простая** — центр в одном руководителе.
- **Функциональная** — по функциям.
- **Дивизиональная** — по продукту/рынку/региону.
- **Матричная** — двойное подчинение (функция + проект).
- **Конгломерат** — набор разных бизнесов.
- **Виртуальная** — ядро + внешние партнёры.

---

## Две экстремальные модели оргструктуры
- **Механистическая** — жёсткая иерархия, формализация, централизация.
- **Органическая** — гибкость, децентрализация, команды.
На практике — гибрид.

---

## Существующие ИС
Перечень действующих систем и их функций.  
Нужно для интеграции и выявления дублирования.

---

## Техническая оснащённость
Оборудование, сети, рабочие места.  
Оценивается готовность инфраструктуры.

---

## Технологические процессы
Последовательность операций выполнения работ.  
Описываются операции и используемые средства.

---

## Материальные потоки
Движение материалов/продукции по этапам.  
Фиксируются маршруты и точки контроля.

---

## Бизнес-процессы
Процессы верхнего уровня (As Is).  
Бывают: управленческие, основные, обеспечивающие.

---

## Классификатор «Подразделение – процесс – функция»
Показывает, кто за что отвечает.  
Используется для формирования требований.

---

## Узкие места
Проблемы процессов или ИС.  
Либо устраняются, либо фиксируются как риски.

---

## Глоссарий
Словарь терминов проекта.  
Нужен для единого понимания.

---

## As Is / To Be
- **As Is** — текущее состояние.
- **To Be** — целевое состояние после ИС.

---

Основа для:
